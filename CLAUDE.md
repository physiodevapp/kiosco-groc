# CLAUDE.md ‚Äî Kiosco GROC

## What this project is

Touch kiosk for recording the Global Rating of Change (GROC) scale at discharge of rehabilitation patients. The patient moves a slider from -7 to +7 and the value is saved to Google Sheets.

## Stack

- `C√≥digo.js` ‚Äî Google Apps Script (backend): exposes `doGet()` and `guardarRespuesta(datos)`
- `index.html` ‚Äî Full frontend (HTML + CSS + JS inline), served by GAS via `HtmlService`
- `appsscript.json` ‚Äî GAS project configuration (timezone, permissions, V8 runtime)
- `.clasp.personal.json` / `.clasp.prod.json` ‚Äî scriptIds for each environment (see Deploy flow)
- `.clasp.json` ‚Äî active file generated by the Makefile, ignored by git

## Key architecture

- No custom server or database: everything runs on Google Apps Script
- Frontend‚Üíbackend communication is via `google.script.run` (only works when the page is deployed on GAS, not on Live Server)
- The Google Sheets `SHEET_ID` **is not in the code** ‚Äî it's stored in GAS script properties (`PropertiesService.getScriptProperties()`)
- Tailwind CSS v2 via jsDelivr CDN (no build step)

## Important development limitation

`google.script.run` does not exist outside GAS. Live Server is useful for iterating the UI, but to test data submission you need to `clasp push` and open the webapp URL.

## Deploy flow

Two environments managed with `make`. The `.clasp.json` file is generated automatically and is not versioned.

```bash
make push-personal   # pushes to personal GAS project (testing)
make push-prod       # pushes to production GAS project and reverts .clasp.json
```

For changes to be visible at the public URL it may be necessary to create a new deployment in GAS (`clasp deploy`).

## Validations

- `fechaFin`: `YYYY-MM-DD` string, automatically generated on the client as today's local date
- `idProceso`: regex `A\d{6}` ‚Äî always "A" + 6 digits (e.g. A000001), built by the numpad
- `valorGroc`: integer between -7 and 7
- Validation on both client (`index.html`) and server (`C√≥digo.js`)

## Commit format

When asked to make a commit, always use this format:

```bash
git commit -m "short imperative title" -m "description when necessary"
```

- The first `-m` is the title (max ~72 characters)
- The second `-m` is only included when there is relevant context to add
- Never use `git commit` without flags or interactive editors
- **Do not add co-authorship** (`Co-authored-by`) under any circumstances

## Security

- `.clasprc.json` (Google OAuth token) excluded by `.gitignore` ‚Äî lives in `~/.clasprc.json`
- `.clasp.json` excluded by `.gitignore` ‚Äî generated by the Makefile, contains no credentials
- `.clasp.personal.json` / `.clasp.prod.json` ‚Äî versioned, contain only the scriptId
- LockService on the backend to prevent concurrent writes to Sheets
